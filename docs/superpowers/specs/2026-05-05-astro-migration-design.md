# Astro Migration Design Spec

**Date:** 2026-05-05
**Project:** ricki-web portfolio
**Migration:** Vue 3 + Vite + vite-ssg → Astro 5 (full rewrite, Option C)
**Strategy:** Page-by-page sequential (Option A)

---

## 1. Overview

Full rewrite of the portfolio from Vue 3 to Astro 5. No Vue integration layer — all components become `.astro` files or React islands for interactive UI. The migration preserves all existing pages, routes, and features while gaining Astro's built-in SSG, file-based routing, Content Collections for blog/projects, and zero-JS-by-default performance.

**Goals:**

- Full static output (SSG) deployable to Vercel/Netlify/Cloudflare Pages
- Zero `any`, zero `console`, zero `unknown` — strict TypeScript throughout
- SEO-complete: sitemap, robots.txt, OG tags, JSON-LD, canonical URLs per page
- PR automation via DangerJS
- Unified lint + format via Biome (`.ts`/`.tsx`/`.js`) + eslint-plugin-astro (`.astro`)

## 2. Architecture

### Framework Layer

| Concern            | Vue 3                         | Astro 5                                          |
| ------------------ | ----------------------------- | ------------------------------------------------ |
| Routing            | vue-router                    | File-based (`src/pages/`)                        |
| State              | pinia                         | nanostores                                       |
| SSG                | vite-ssg                      | Astro built-in                                   |
| Head/SEO           | @unhead/vue                   | `BaseHead.astro` component                       |
| UI primitives      | reka-ui + shadcn-vue          | plain `.astro` + shadcn/ui (React)               |
| Interactive UI     | Vue SFCs                      | React islands (`client:load` / `client:visible`) |
| Icons              | lucide-vue-next               | lucide-react (React islands)                     |
| Animations         | @vueuse/motion                | motion (vanilla JS API)                          |
| Complex animations | motion                        | GSAP (scroll-triggered)                          |
| Form validation    | vee-validate + zod            | zod (client-side) + Web3Forms                    |
| Toast              | vue-sonner                    | sonner (React island)                            |
| Hand gesture       | @mediapipe/tasks-vision (Vue) | @mediapipe/tasks-vision (Web Component)          |
| Syntax highlight   | shiki                         | shiki (native Astro support)                     |
| Font manipulation  | opentype.js (Vue)             | opentype.js (vanilla `<script>`)                 |
| Theme toggle       | pinia store                   | nanostores                                       |

### Integration Layers Added to Astro

- `@astrojs/react` — required for shadcn/ui, sonner, interactive components
- `@astrojs/tailwind` — Tailwind CSS integration
- `@astrojs/sitemap` — auto-generates `sitemap.xml`
- `astro-robots-txt` — generates `robots.txt`

## 5. Linting & Code Quality

### Biome (`biome.json`) — handles `.ts`, `.tsx`, `.js`

```json
{
  "linter": {
    "enabled": true,
    "rules": {
      "recommended": true,
      "suspicious": {
        "noExplicitAny": "error",
        "noConsole": "error"
      },
      "correctness": {
        "noUnusedVariables": "error",
        "noUnusedImports": "error"
      },
      "style": {
        "noNonNullAssertion": "warn"
      }
    }
  },
  "formatter": {
    "enabled": true,
    "indentStyle": "space",
    "lineWidth": 100
  },
  "javascript": {
    "formatter": {
      "quoteStyle": "single",
      "trailingCommas": "none",
      "semicolons": "asNeeded"
    }
  }
}
```

### eslint-plugin-astro (`.eslintrc-astro.cjs`) — handles `.astro` only

```js
module.exports = {
  overrides: [
    {
      files: ['*.astro'],
      parser: 'astro-eslint-parser',
      extends: ['plugin:astro/recommended'],
      rules: {
        'astro/no-unused-define-vars-in-style': 'error',
        'astro/no-conflict-set-directives': 'error'
      }
    }
  ]
}
```

### TypeScript (`tsconfig.json`) — strict mode

```json
{
  "compilerOptions": {
    "strict": true,
    "noImplicitAny": true,
    "useUnknownInCatchVariables": true,
    "noUnusedLocals": true,
    "noUnusedParameters": true
  }
}
```

### Lint-Staged (`.husky/pre-commit`)

```
biome check --write → for .ts/.tsx/.js
eslint --fix        → for .astro files
```

### DangerJS (`dangerfile.ts`)

- Warn on PRs with diff > 500 lines (suggest splitting)
- Enforce conventional commit format on PR title
- Require at least one test file change if `src/` is modified
- Fail if `console.log` appears in diff

## 6. SEO Strategy

Every page uses `BaseHead.astro` which injects:

- `<title>` + `<meta name="description">`
- Open Graph tags (`og:title`, `og:description`, `og:image`, `og:url`)
- Twitter Card meta
- `<link rel="canonical">`
- `<html lang="en">`
- JSON-LD structured data (Person schema on Home/About, BreadcrumbList on detail pages)

Additional SEO:

- `@astrojs/sitemap` auto-generates `/sitemap.xml` from all static routes
- `astro-robots-txt` generates `/robots.txt`
- Astro `<Image>` component for all images (WebP conversion, `width`/`height`, lazy loading)
- Preload hints for critical fonts

## 7. Page Migration Order

### Phase 0 — Bootstrap

- Init Astro 5, configure integrations, Biome, eslint-plugin-astro, husky, dangerjs
- `BaseHead.astro`, `MainLayout.astro`, `AppHeader.astro`, `AppFooter.astro`
- nanostores theme store, global CSS, tsconfig strict mode

### Phase 1 — Static Pages

1. `/experience` — pure static, no interactivity
2. `/about` — SkillsMatrix + SkillsTimeline (`.astro`), GitHubHeatmap (React island)

### Phase 2 — Content-Driven

3. `/projects` + `/projects/[slug]` — Content Collection, ProjectFilters React island
4. `/blog` (coming soon) + `/blog/[slug]` — Content Collection MDX, placeholder post

### Phase 3 — Interactive

5. `/contact` — ContactForm React island (zod + Web3Forms)
6. `/works` — HandGestureDetector Web Component

### Phase 4 — Home

7. `/` — all home components, SignatureSpill, animations

### Phase 5 — Polish

- Motion + GSAP animation pass
- Full SEO audit per page
- Performance: Astro Image, lazy loading, preload hints
- `404.astro`
- Zero lint errors, zero TS errors

## 8. Key Constraints

- **No `any`** — Biome enforces `noExplicitAny: error`. Use `unknown` + type guards.
- **No `console`** — Biome enforces `noConsole: error`. Use structured logging util if needed.
- **No `unknown` as implicit** — `useUnknownInCatchVariables: true` in tsconfig.
- **React islands use `client:visible`** by default (lazy hydration). Use `client:load` only for above-the-fold interactive elements.
- **Blog** starts as MDX Content Collections, designed to accept CMS data later by swapping the collection loader.
- **Works page** hand gesture feature is a native Web Component — no framework dependency.
- **Package manager:** keep `bun`.
